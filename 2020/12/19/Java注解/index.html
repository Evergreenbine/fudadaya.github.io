Java注解的基本使用

从JDK5开始，Java增加了对Annotation的支持，就是注解。那注解是什么呢？我们可以理解为一个标记，用来添加一些额外的辅助信息。不同于注释的是，注释是会被编译器忽略，但注释可以在编译，类加载，运行时被读取，执行相应的处理。

内置注解

Java中有5个基本的内置注解

  • @Override: 代表这是一个重写的方法
  • @Deprecated: 定义在java.lang.Deprecated中，此注释可以用于修辞方法，属性，类，表示不鼓励程序员使用这一的元素，或者有更好的代替，就平常会遇到一些方法有横线划过去，就是加了这个注解。
  • @SuperpressWarning: 压制警告，有些方法或者变量会报一些警告，加上这个注解，则会抑制编译时的警告信息
  • @SafeVarargs: 主要目的是处理可变长参数中的泛型，此注解告诉编译器：在可变长参数中的泛型是类型安全的。可变长参数是使用数组存储的，而数组和泛型不能很好的混合使用
  • @FunctionalInterface: 代表这是一个函数式接口

元注解

元注解的作用就是负责注解其它的注解，java中定义了4个标准的元注解类型

  • @Target：用于描述注解的使用范围，比如说定义在方法上，定义在类上

  • @Retention：表示需要在什么级别中保存该注释信息，用于描述注解的在什么时期生效

它们描述的时期范围包含是 RUNTIME > CLASS > SOURCE,即RUNTIME生效即表明他在SOURCE生效。

  • @DOCUMENT：说明该注解将被包含在javadoc中

  • @Inherited：说明子类可以继承父类中的注解

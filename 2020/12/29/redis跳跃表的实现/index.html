跳跃表概述

下面举一个和跳跃表最为贴切的例子，就是我们常见到的目录。如果我们不添加一级目录，二级目录这样的多级目录的话，我们要查找我们想要获得的内容时，就不得不从头开始像向尾找。所以为了更快的找到我们想要的内容，就会在一些节点上添加一级更高层级的目录。这样我们就可以先查找最高级目录，再往下找我们需要的内容。讲到这，是不是好像感觉这个听起来牛逼哄哄的数据结构也没那么niubility哈哈？下面我们简单介绍一下跳跃表，关于它的实现会在数据结构分类中专门写一篇文章介绍它。

Redis里的跳跃表

上图展示的就是redis里跳跃表的大致结构，它包含有下面的一些属性

struct zskiplistNode *header：指向跳跃表的表头节点。

struct zskiplistNode *tail：指向跳跃表的表尾节点。

int level：记录目前跳跃表内，层数最大的那个节点的层数（表头节点的层数不计算在内）。

unsigned long length：记录跳跃表的长度，也即是，跳跃表目前包含节点的数量（表头节点不计算在内）。

而它的节点类型的结构体如下，下面我们介绍下它节点类型的属性

double score：上图中的1.0,2.0，3.0等分值，节点按各自所保存的分值从小到大排列

struct zskiplistNode *backward：后退指针，即如果在一个高level中，遍历寻找到一个比要查找的值大的节点时，会回退到这个节点的上一个节点，然后就会往下层级去遍历，依次循环直到找到值或到null。

struct zskiplistLevel level[] :节点的层级数组，每个层级包含一个forward前进指针，一个span跨度，跨度的意思就是两个节点之间的距离。
